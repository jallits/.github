# Contributing

Thank you for your interest in contributing.

## Getting Started
- Fork the repository
- Create a feature branch from `master`
- Make focused, well-documented changes

## Issues
- Search existing issues before opening a new one
- Provide clear reproduction steps for bugs
- Include context and rationale for feature requests

## Pull Requests
- Keep PRs small and focused
- Link related issues
- Ensure tests and checks pass
- Use clear commit messages that are signed

## Code Style
Follow existing conventions and formatting within the repository.
